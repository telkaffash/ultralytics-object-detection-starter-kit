# Object Detection Starter Kit (Ultralytics)

A clean, **beginner-friendly**, repo for training and comparing **object detection models** on custom datasets (using Ultralytics).

The example dataset was developed during a senior design project on autonomous date pollination at Qatar University, and is available on:
> **Demo dataset:** [Female Date Flowers Dataset](https://universe.roboflow.com/date-female-flowers/date-female-flowers-updated-y0hjq)


## Models Supported

| Script | Model | Size | Best For |
|---|---|---|---|
| `train_yolov8n.py` | YOLOv8n (nano) | ~6 MB | Fast experiments, edge devices |
| `train_yolov8s.py` | YOLOv8s (small) | ~22 MB | Best accuracy–speed balance |
| `train_rtdetr_l.py` | RT-DETR-l (large) | ~134 MB | Highest accuracy, transformer-based |

## Repo Structure
```
├── scripts/
│   ├── train_yolov8n.py      ← Train YOLOv8 nano
│   ├── train_yolov8s.py      ← Train YOLOv8 small
│   ├── train_rtdetr_l.py     ← Train RT-DETR large
│   └── compare_models.py     ← Compare all trained models
├── dataset/                  ← Put your downloaded dataset here
│   └── data.yaml             ← Auto-generated by Roboflow
├── results/                  ← CSVs with training metrics (auto-created)
├── runs/                     ← Ultralytics training outputs (auto-created)
├── requirements.txt
└── README.md
```

## Quickstart - How to use?

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

> **GPU users:** install PyTorch with CUDA support first - see [pytorch.org](https://pytorch.org/get-started/locally/).
---

### 2. Get your dataset

Download your dataset or label your own images using **Roboflow**, **CVAT**, **LabelImg**, or any tool that exports in **YOLOv8 format**. I recommend [Roboflow](https://universe.roboflow.com/).

When exporting:
- Select **YOLOv8** as the export format
- Download and unzip into the `dataset/` folder

Your `dataset/` folder should look like this:

```
dataset/
├── data.yaml          ← class names + split paths
├── train/
│   └── images/
│   └── labels/
├── valid/
│   └── images/
│   └── labels/
└── test/
    └── images/
    └── labels/
```
Ensure all image files use standard formats such as .png, .jpg, or .jpeg, and make sure it's not image.png.png. Yes, this happens more often than you'd expect.
---


### 3. Update the data path

Open whichever training script you want to use and set `DATA_YAML`:

```python
# In scripts/train_yolov8s.py (or any other script)
DATA_YAML = "dataset/data.yaml"   # ← already set correctly if you did step 2
```

---

### 4. Train a model

Pick any model and run its script:

```bash
# YOLO nano
python scripts/train_yolov8n.py

# YOLO small
python scripts/train_yolov8s.py

# RT-DETR large
python scripts/train_rtdetr_l.py
```

Training will:
- Download pre-trained weights automatically (first run only)
- Save checkpoints to `runs/detect/<model_name>/weights/`
- Print live training progress (loss, mAP per epoch)
- Evaluate on your validation split when done
- Save a metrics CSV to `results/`

---

### 5. Compare models

After training two or three models, run:

```bash
python scripts/compare_models.py
```

This produces `results/model_comparison.csv` with:

| Column | Description |
|---|---|
| `mAP@0.5` | Standard detection accuracy metric |
| `mAP@0.5:0.95` | Stricter accuracy (averaged over IoU thresholds) |
| `Precision` | Of all predicted boxes, how many were correct |
| `Recall` | Of all real objects, how many were found |
| `Latency mean (ms)` | Average inference time per image |
| `FPS` | Frames per second |
| `Size (MB)` | Model file size |
---

## Tuning Tips

| Parameter | Where to change | Effect |
|---|---|---|
| `EPOCHS` | top of each training script | More epochs → higher accuracy (up to a point) |
| `BATCH` | top of each training script | Lower if you get out-of-memory errors |
| `IMG_SIZE` | top of each training script | Higher → more detail, slower training |
| `DEVICE` | top of each training script | `0` for GPU, `"cpu"` for CPU |
